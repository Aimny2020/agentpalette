# Skills 自定义技能说明设计

## 1. 目标

为技能包和每个子 Skill 增加一个由用户手动维护的“技能说明”字段，用于以中文快速说明技能用途。

自定义技能说明必须与第三方 Skill 源文件隔离，不修改 `SKILL.md`，也不得在技能更新、重新扫描或覆盖安装时被覆盖。该数据在当前 AgentForge 用户范围内全局共享，并支持通过 JSON 备份和跨设备迁移。

## 2. 范围

第一版包含：

- 为技能包和每个子 Skill 分别维护一条纯文本技能说明。
- 在技能目录、详情页和项目 Skills 启用页面展示技能说明。
- 使用技能说明参与搜索。
- 使用版本化 JSON 导出、预览并导入技能说明。
- 保存和管理当前没有对应已安装 Skill 的未关联说明。

第一版不包含：

- AI 自动翻译或自动总结。
- 修改 `SKILL.md` 或技能目录中的其他文件。
- Markdown、HTML 或富文本说明。
- 中文名称、使用示例、注意事项等额外结构化字段。
- 多语言字段。
- 项目级技能说明覆盖。
- 云同步。
- 基于技能名称的模糊关联。
- 分类、用户备注、信任状态或项目启用状态的备份。

## 3. 核心概念

### 3.1 字段定义

界面名称统一为“技能说明”，内部字段名为 `custom_description`。

技能说明是用户编写的纯文本，最大长度为 2,000 个 Unicode 字符。空白字符串不作为有效说明保存。

### 3.2 目标身份

每条说明通过稳定的 `target_id` 关联目标：

- 技能包或独立 Skill：使用顶层 `skill.id`，例如 `taste-skill`。
- 技能包成员：使用现有成员 ID，即包 ID 与成员相对目录组合，例如 `taste-skill::skills/minimalist-ui`。

`target_kind` 只能是：

- `package`：顶层技能包或独立 Skill。
- `member`：技能包内的具体 Skill。

`target_id` 是关联依据。技能名称、英文描述和作者信息不得用于自动关联。

## 4. 持久化规则

### 4.1 运行时数据源

SQLite 是唯一运行时数据源。新增独立的 `skill_descriptions` 表，不将说明并入现有包级 `skills_user_meta`，也不以 JSON 字段保存子技能集合。

建议的数据结构：

```sql
CREATE TABLE skill_descriptions (
    target_id TEXT PRIMARY KEY NOT NULL,
    target_kind TEXT NOT NULL CHECK (target_kind IN ('package', 'member')),
    custom_description TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

该表不对当前已安装 Skill 建立外键。这样才能在技能卸载后保留说明，并在相同 ID 的技能重新安装时恢复关联。

### 4.2 写入规则

- 保存非空说明时，对 `target_id` 执行新增或更新，并刷新 `updated_at`。
- 用户清空说明并保存时，删除对应数据库记录。
- 单条保存必须使用事务。
- 保存失败时保留编辑框内容、显示明确错误并保持详情页打开。
- 自定义说明只能由明确的用户编辑或导入操作修改。

### 4.3 生命周期规则

- 技能扫描只读取说明并合并到返回模型，不得改写说明。
- 技能更新、覆盖安装及 Git 版本变化不得改写或删除说明。
- 普通卸载不得删除说明；对应记录转为未关联状态。
- 重新安装相同 `target_id` 的技能后，说明自动重新关联。
- 子 Skill 路径变化会产生新的 ID。旧说明继续保留为未关联记录，不进行名称相似度匹配或自动迁移。
- 用户可以从管理入口手动清理未关联说明；执行前必须展示记录数量并要求确认。

## 5. 读取与展示规则

### 5.1 回退顺序

所有需要简短描述的界面使用同一条规则：

1. `custom_description` 存在且非空时，显示自定义技能说明。
2. 否则显示原始 `metadata.description`。

自定义说明旁显示轻量“自定义”标识，以明确区分用户内容与技能作者提供的内容。

### 5.2 展示位置

- Skills 管理页的技能包卡片。
- 搜索结果中的子 Skill 卡片。
- 技能包详情中的子 Skill 卡片。
- 项目 Skills 启用页面中的技能包及子 Skill。

技能详情中的 `SKILL.md` 正文继续按原样展示。自定义技能说明不替换、不翻译正文。

### 5.3 安全规则

- 自定义技能说明按纯文本渲染。
- 不解析 Markdown。
- 不允许通过 HTML 注入界面内容。
- 单条说明数据异常不得阻止 Skills 列表加载；界面应回退到原始英文描述。

## 6. 编辑规则

技能详情侧栏增加独立的“技能说明”编辑框：

- 打开顶层技能包或独立 Skill 时，编辑顶层目标的说明。
- 进入某个子 Skill 详情时，编辑当前子 Skill 的说明。
- 保存操作只更新当前正在查看的目标。
- 切换目标前如存在未保存修改，应阻止静默丢失，并提示用户保存或放弃。
- 原有“技能使用说明与备注”继续作为包级私人备注，不与技能说明合并，也不改变其既有语义。

第一版不提供批量编辑和自动生成说明。

## 7. 搜索规则

Skills 目录搜索同时匹配：

- 技能名称。
- 自定义技能说明。
- 原始英文描述。

匹配行为：

- 顶层技能信息匹配时，返回顶层技能卡片。
- 子 Skill 信息匹配时，返回对应子 Skill 卡片。
- 自定义说明存在时仍保留对原始英文描述的搜索能力，避免覆盖展示后丢失原始关键词检索。
- 搜索比较沿用当前不区分大小写的行为。

## 8. 导出规则

JSON 仅作为备份和迁移格式，不作为运行时存储。

导出格式必须包含模式版本：

```json
{
  "schema_version": 1,
  "exported_at": "2026-07-07T12:00:00Z",
  "descriptions": [
    {
      "target_id": "taste-skill::skills/minimalist-ui",
      "target_kind": "member",
      "description": "用于设计简洁、偏编辑风格的前端界面。",
      "updated_at": "2026-07-07T11:00:00Z"
    }
  ]
}
```

导出要求：

- 默认导出全部技能说明。
- 必须包含当前未关联到已安装 Skill 的记录。
- 不导出分类、用户备注、信任状态、来源信息或项目启用状态。
- 记录顺序应稳定，建议按 `target_id` 排序，以便审阅和版本比较。

## 9. 导入与冲突规则

### 9.1 校验

写入前必须整体校验：

- `schema_version` 是否受支持。
- `target_id` 是否非空且格式合法。
- `target_kind` 是否为 `package` 或 `member`。
- `description` 是否为非空纯文本且未超过 2,000 个 Unicode 字符。
- `updated_at` 是否为合法时间。
- 同一文件中是否存在重复 `target_id`。

空白说明是无效导入记录，不作为删除指令。

### 9.2 预览

导入必须先展示预览，再由用户确认。预览至少显示：

- 新增数量。
- 覆盖数量。
- 跳过数量。
- 未关联数量。
- 无效记录及原因。

### 9.3 冲突策略

默认策略为“保留较新记录”，比较导入记录和本地记录的 `updated_at`。

用户还可以在本次导入中选择：

- 保留本地：冲突记录全部跳过。
- 使用导入文件：冲突记录全部覆盖。

导入当前未安装技能的说明是合法操作；记录正常保存并标记为未关联。

### 9.4 原子性

- 文件必须完整校验后才能进入写入阶段。
- 批量导入使用单个事务。
- 任意数据库写入失败时整体回滚，不允许部分导入。

## 10. 错误处理

- 保存错误必须包含可操作的信息，但不得泄露敏感系统细节。
- 无法读取某条说明时，当前技能仍应正常展示，并回退到原始英文描述。
- `target_kind` 与当前扫描到的目标类型不一致时，该记录视为无效关联，不自动修正。
- 不支持的导出模式版本必须拒绝导入，并提示当前支持的版本。
- 导出失败不得影响数据库中的现有说明。

## 11. 测试要求

### 11.1 Rust 与数据库

- 包级和子 Skill 说明可以独立保存、读取、更新和删除。
- 清空说明会删除记录。
- 技能更新不改变说明。
- 卸载技能后说明保留。
- 重新安装相同 ID 后说明恢复关联。
- 子 Skill 路径变化时不发生基于名称的错误关联。
- 超长说明和非法 `target_kind` 被拒绝。
- 导入冲突策略符合选择。
- 无效文件不会产生任何写入。
- 批量导入写入失败时整体回滚。

### 11.2 前端

- 有自定义说明时优先显示并出现“自定义”标识。
- 没有自定义说明时显示原始英文描述。
- 包级与子 Skill 编辑器保存各自目标。
- 保存失败时保留输入内容且不关闭详情页。
- 切换目标时不会静默丢失未保存内容。
- 搜索可以匹配中文说明和被覆盖展示的原始英文描述。
- 项目 Skills 启用页面使用同一回退逻辑。
- HTML 字符串按普通文本展示。
- 超过长度限制时无法提交。
- 导入预览正确展示新增、覆盖、跳过、未关联和无效数量。

## 12. 验收标准

满足以下条件即可认为第一版设计目标完成：

- 用户可以为任意已安装的顶层 Skill 或子 Skill 编写一条中文技能说明。
- Skills 目录和项目启用页面优先展示该说明，未填写时可靠回退英文描述。
- 技能更新或卸载不会造成说明丢失。
- 搜索能够使用中文说明定位 Skill。
- 用户能够通过版本化 JSON 完整备份并迁移所有说明，包括未关联记录。
- 所有导入写入具备校验、预览、冲突处理和事务回滚能力。
- 原始 Skill 文件和说明文档始终保持不变。
